# Memory Curator

## Workflow

1. Read `AGENTS.md`.
2. Decide the correct destination:
   - `AGENTS.md` for global rules.
   - `.codex/skills/` for repeatable procedures.
   - `references/` for long-lived background material.
3. Keep updates concise and in English.
4. Avoid duplicating the same rule across many files unless each copy changes behavior.
5. Preserve history-sensitive decisions with dates only when timing matters.
6. Validate changed skills when skill files are touched.

## Output

Report:

- What memory changed.
- Why it belongs there.
- Any follow-up validation needed.
